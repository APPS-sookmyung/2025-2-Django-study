> HTML

* 웹은 프론트엔드와 백엔드로 구성
  * 프론트: 보여지는 부분 디자인적 요소/ 백엔드: 프론트 요청 내용 처리

* HTML: 하이퍼텍스트 마크업 언어

> HTML 문법 작성 규칙
* 태그 구조: 시작 태그와 종료 태그가 한 묶음, 종료 태그는 </>
* HTML5 문서의 시작 <!Doctype html>, 웹 문서 내용 시작 <html></html>
* 주석: <!--내용-->* 으로 표기

> HTML 태그 모음
1. <h1~6>: 제목 구분, 숫자 커질 수록 작은 크기
2. <p>문자</p>: 단락 변경
3. <br>: 개행
4. <meata charset="UTF-8">: 인코딩 형식 설정
5. <a href="링크">내용</a>: 하이퍼링크 삽입
6. <table>: 행과 열이 바둑판 형태로 구성된 표 만들기
7. <div></div>: 다른 엘리먼트를 감싸 그룹 정의
8. <form>
   1. name 폼 이름 지정
   2. action 입력 결과 전달 받을 서버 URL
   3. method 입력 결과 전달 방법
9.  <input>: 입력 태그 

> HTML 이미지 삽입하기
* <img> 이미지 삽입하는 태그
* 주요 속성
  * src: 이미지의 경로
  * alt: 이미지를 표시할 수 없을 때 출력 내용
  * width: 이미지 가로크기
  * height: 이미지 세로크기
  * loading: 이미지 가져오는 방식
* 예
  * <img src="" width="" height="" loading="">

>HTML 표 만들기
* <table></table> :표를 만들 때 사용하는 태그
* 행과 열이 바둑판 형태로 구성
  * <tr>행
  * <td>셀 내용
  * <th>헤더 셀
  * <thead>도표 머리행
  * <tbody>도표 실제 내용
  * <tfoot>도표 마지막 행
* 예
```
  <table>//전체 표
    <tr>//행설정

> HTML 입력 받기

* input :정보를 입력받는 태그(마감태그 없음)
    ```
    <input type="text" value="초기값" name="이름"/>
    ```
* form: 입력한 정보를 서버에 전송하는 태그
    ```
    <form action="url>
        name: <input type="text" name=""><br>
        <input type="submit" value="Submit"><br>
    </form>
    ```
* input 태그 요소
  * type="" : 지정한 형태로 입력을 받겠다
  * value : 실제 적힌 값
  * autofocus : 커서를 자동으로 깜빡임
  * autocomplete : 자동완성 기능 지원 여부 지정
  * placeholder : 안내 글 
  * readonly : 필드를 읽을 수만 있게
  * maxlength : 필드 최대문자개수
  * minlength : 필드 최소문자개수
  * required : 반드시 값이 있어야 제출 가능
* 다양한 입력 방식
  * text : 기본 문자열 입력 창 생성
  * <textarea></textarea> : 여러 줄을 받는 태그, cols: 가로, rows: 세로
  * password : 비밀번호 입력 창 생성
  * button : 기본 버튼
  * radio : 라디오 버튼
  * checkbox : 체크박스 생성
  * file : 파일 선택 버튼 생성
  * image : 이미지를 버튼처럼 사용 
  * hidden : 기본으로 설정 값을 서버로
  * submit : 서버로 제출하는 버튼 생성
  * reset : form에서 입력한 input값을 초기화
  * search : X 를 누르면 입력한 값이 다 사라짐
  * range : 범위 설정
  * email : 이메일 정규식 
* 폼 태그 제출
  * action="url": 입력 제어 처리 url 주소 지정
  * method: method 제출 http 타입 

> HTMl 공간 분할
>
* HTML 모든 요소는 display 속성을 가짐
* block: p, div, h, ul, ol, form

<h2>웹 페이지 서식 CSS 이해하기</h2>

> CSS
* HTML 표현 방법 기술
* 아무리 좋은 기능을 가진 웹 페이지라도 디자인이 좋지 못하면 구려보임 
* 디자인 요소 분리 정의
* 디자인 재사용 지원
* 반응형 디자인이 가능해짐 

> CSS 기본 작성 문법

* 선택자와 선언부로 구성
  * 선택자: 스타일을 지정할 HTML 태그 요소
  * 선언부: CSS 속성 이름과 값의 쌍 형태 (딕셔너리)
* 선언부: 콜론으로 구분된 다수 항목 포함, 선언 후 세미콜론으로 종결
  * 선언부 중괄호로 블록 단위 구분
    ```
    h3{color: blie; font-szie: 16px;}
    ```

> CSS 적용 방법
>
* CSS 적용 방법은 내부, 외부, 인라인 스타일 구성이 있음
* 내부: HTML 내부에 스타일 기술
* 외부: .css 파일에 내용 작성 후 HTML 문서에서 호출
* 인라인: 각 HTML 태그에 필요한 디자인 속성 직접 정의

> CSS 셀렉터 지정
* 셀렉터: 서식 지정 대상 선택
  * 기본적으로 태그,아이디, 클래스를 셀렉터로 사용
  
> CSS 서식 모음

   * color: 색
   * text-align: 문자열 정렬
   * font-family: 글꼴
   * font-style: 이탤릭체, normal 등
   * font-size: 폰트 크기
   * font-weight: 폰트 두께 지정
  
> CSS 셀렉터 조합
* 기본 셀렉터를 조합해 다양하게 셀렉터 정의 가능
  * 동시 지정
    ```
    #login-form input, #login-form label{display: block;}
    ```
  * 태그와 클래스 결합
    ```
    .header{color: red;}
    h1.header{color: blue;}
    h2.header{color:green;}
    ```
  * 폼 지정
    ```
    #login-form input[type="submit"]{margin:10px auto;}
    ```

* 웹사이트를 쉽게 만들 수 있게 도와주는 프론트엔드 프레임워크

* 웹 사이트 디자인 소요 시간 단축 및 반응형 웹을 위해 등장하였음

> 부트스트랩 사용방법

1. 부트스트랩 파일을 내려 받아 불러오기
2. 부트스트랩 CDN 적용
3. https://getbootstrap.com/docs/4.5/getting-started/introduction/
   1. CDN?:contents delivery network 지리적 물리적으로 떨어진 사용자에게 컨텐츠를 빠르게 제공하는 기술
   2. 목적: 높은 사용성과 효율로 컨텐츠를 전달
   3. CDN을 사용하지 않으면 기존 서버가 모든 요청에 일일이 답해야하지만 CDN은 (각 나라에 캐시 서버가 존재하기 때문에 좀 더 빠른 응답 가능)기존을 대신해 응답할 수 있음
